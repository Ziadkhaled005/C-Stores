# Contributing

Thanks for contributing! A few guidelines to get started:

- Branching: create feature branches from `main` or `master` with descriptive names.
- Commits: keep commits small and focused; prefer present-tense messages.
- Code style: follow the existing code style. Run linters/formatters before submitting.
- Tests: add tests for new behavior where applicable.
- Pull requests: include a short description of what changed and why, and link related issues.
