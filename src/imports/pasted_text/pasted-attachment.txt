2. MULTI-BRANCH INVENTORY SYSTEM
================================

Convert the inventory system into a Multi-Branch ERP.

Create 5 independent branches.

Example:

* الفرع الرئيسي
* فرع 1
* فرع 2
* فرع 3
* فرع 4

The number of branches must be configurable.

=================================================
BRANCH ISOLATION
================

By default:

Users can ONLY access their assigned branch.

They must NOT see:

* Inventory of other branches
* Sales of other branches
* Purchases of other branches
* Customers of other branches
* Reports of other branches
* Financial information of other branches

All branch data must be isolated.

=================================================
BRANCH ACCESS CONTROL
=====================

System Administrator can define:

* Single Branch Access
* Multiple Branch Access
* All Branch Access

Examples:

Cashier → Branch 1 only

Sales Manager → Branch 1 + Branch 2

General Manager → All Branches

=================================================
INVENTORY BY BRANCH
===================

Each branch should have its own:

* Stock Quantity
* Purchase Records
* Sales Records
* Inventory Movements
* Transfers

Product catalog remains shared globally.

Example:

Dell Latitude 5540

Branch 1 = 20 Units
Branch 2 = 15 Units
Branch 3 = 5 Units
